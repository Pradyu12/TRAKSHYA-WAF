# Security Policy

## Reporting a Vulnerability
Please do not report security vulnerabilities through public GitHub issues.

Instead, please report them via email to the maintainers at the contact listed in the repository profile or README.

Include:
- Description of the vulnerability
- Steps to reproduce
- Affected versions or commits
- Suggested remediation, if any

We aim to acknowledge reports within a few days and provide a more detailed response within a reasonable timeframe.

## Supported Versions
Use the latest release when possible.

| Version | Supported |
|---------|-----------|
| latest  | yes       |
| older   | no        |

## Security Updates
Security advisories will be published in the repository Security tab once verified and patched.
