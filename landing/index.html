SIEM Correlation

7 detection rules: brute force, port scanning, XSS/SQLi waves, geo anomaly, beaconing, data exfiltration, lateral movement.

Rules (MITRE ATT&CK):
T1110 Brute Force — Failed auth thresholding
T1046 Port Scan — SYN/connect scan detection
T1190 XSS/SQLi Wave — Burst pattern correlation
T1071 Beaconing — Periodic C2 detection
T1041 Data Exfil — Large outbound transfers
T1021 Lateral Movement — Internal scanning
T1190 Geo Anomaly — Impossible travel detection
⚠

HIDS + FIM + SCA

Host-based intrusion detection via auth log parsing, SHA-256 file integrity monitoring, and CIS benchmark compliance checks.

HIDS: /var/log/auth.log, journalctl — SSH brute force, sudo abuse, new users
FIM: SHA-256 baseline of /etc/passwd, /etc/shadow, /etc/ssh/sshd_config, /bin/*, /sbin/*
SCA (CIS): File perms (644/755), SSH config (PermitRootLogin no), password policy (min 12 chars), kernel params
💡

Active Response

iptables/nftables IP blocking, UFW integration, systemd posture lockdown (Monitor / Standard / Under Attack).

Postures:
Monitor — Log only, 50 req/10s
Standard — Block threats, 50 req/10s
Under Attack — Aggressive, 10 req/10s, auto-block top offenders
Actions: iptables -A INPUT -s $IP -j DROP, nft add rule, ufw deny from $IP
🔍

Rule Engine

Regex-based rule tester with export/import via JSON. Create, test, and deploy custom WAF rules in seconds.

Built-in: OWASP Core Rule Set (CRS) compatible
Categories: SQLi (942100-942180), XSS (941100-941180), RFI (930100), CMDi (932100), Path Traversal (930120)
Testing: Live regex tester with sample payloads, hit counting, false positive tracking
📈

Attack Simulator

8 built-in attack buttons that fire real HTTP requests to test your proxy. See results instantly in the SIEM dashboard.

Attacks: SQLi (union, boolean, time), XSS (reflected, stored, DOM), CMDi (semicolon, backtick, subshell), Path Traversal (../, ..\, URL encoded), RFI, XXE, SSTI, LFI
Target: Configurable upstream (default localhost:8080)
Results: Blocked/Allowed, matched rule, response time, incident ID
🎨

Synthwave UI

Two theme variants (Deep Void / Neon Storm), custom frameless title bar, reactive particles, and fullscreen globe mode.

Themes (7): Cyber Violet, Matrix Green, Neon Cyan, Blood Red, Electric Blue, Synthwave Pink, Toxic Amber
Features: localStorage persistence, keyboard shortcuts (Ctrl+K, 1-9, F, ?), particle background, scanlines
Export: Dashboard state as JSON, incident reports, heatmap data

Technical Deep Dive

Rust Proxy Pipeline

Request → Rate Limit → GeoIP → JWT Auth
      → Rule Engine (SQLi/XSS/RFI/CMDi/LFI/XXE/SSTI)
      → Circuit Breaker → Upstream

Zero-copy hyper pipeline, <1ms p99 latency at 10k req/s. Token bucket per IP (configurable burst). Circuit breaker: 5 failures → half-open → 3 successes → closed.

Go API & SIEM

REST API :8000
├── /api/incidents    CRUD + acknowledge
├── /api/config       Proxy config (YAML)
├── /api/posture      Monitor/Standard/Under Attack
├── /api/agents       Fleet registration
├── /ws / /events     Real-time WebSocket/SSE
└── /metrics          Prometheus exposition

SQLite for incidents/agents/config. 7 SIEM rules with sliding windows (5m-1h). Webhook dispatcher for Slack/Discord. OpenTelemetry OTLP exporter.

C Daemon (System Monitoring)

HIDS    → auth.log / journalctl parsing
FIM     → SHA-256 baseline + inotify
SCA     → CIS Benchmark checks
Vuln    → dpkg/rpm + CVE database
Active  → iptables/nftables/UFW

libmicrohttpd for HTTP reporting to Go API. Runs as root for system access. systemd service with restart=always. Reports every 30s via HTTP POST.

Run Locally in Minutes

No cloud accounts, no API keys, no data leaks. Pick your method.

# One-liner local install (Linux/macOS) $ cd ~ && git clone https://github.com/Pradyu12/TRAKSHYA-WAF.git && cd TRAKSHYA-WAF && bash install.sh

Clones the repo and sets up locally. Builds Rust proxy if available, otherwise uses the mock server. Dashboard at http://localhost:8000.

Runs locally from source. Builds Rust proxy with cargo if available, otherwise falls back to the Node.js mock server. No cloud, no external downloads.

# Use the local npm CLI package $ cd npm-package && npm link # Launch controller $ trakshya-install install --mode=local # Or start directly $ trakshya-waf
